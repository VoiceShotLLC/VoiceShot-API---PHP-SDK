Test Configuration Inbound text message example
 
To test communication between VoiceShot and your Web server:
 
Log into your VoiceShot account.
Open the text messaging campaign named Developer Text Message Demo v5.0.

From the menu bar, click Campaign --> Options

Click the Developer tab.
For the Inbound Call Summary URL, click the Test button.  Your server should respond with 200 OK.
Click Call Summary.  You should see the test Call Summary sent from VoiceShot.
 
The files in this demo write the XML received from VoiceShot to text files in the same directory that these files are in.  Your Web server will need "write permissions" to this directory.
 
Writing files is only part of the demonstration and note required for production.  In a production environment, it is likely that your application will be communicating with a database instead.
